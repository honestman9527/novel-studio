# Genre Modules

按类型选择模块，不要把所有模块都塞进项目。

多类型作品用“主类型 + 辅助类型”组合：主类型决定读者承诺和章节推进，辅助类型只补规则或质感。

## 通用类型契约

无论什么题材，都先写清：

- 读者来读什么：爽点、情绪、谜题、关系、奇观、现实感。
- 主角凭什么持续行动：欲望、创伤、责任、利益、好奇、求生。
- 长线冲突靠什么不断升级：规则、敌人、关系、资源、真相、时间限制。
- 不写什么：禁用套路、尺度边界、设定雷区、叙事视角限制。

## 系统文

写入位置：`novel-studio/memory.yaml` 的 `genre.system`

字段：

- 系统来源：
- 绑定条件：
- 面板字段：
- 任务类型：
- 奖励类型：
- 惩罚/限制：
- 升级曲线：
- 系统不可做的事：
- 与主线冲突的关系：

避免：奖励无代价、系统万能、主角只等任务。

系统文额外检查：

- 系统的存在是否会制造麻烦，而不只是发奖励。
- 成长曲线是否能分阶段变化：新手、破局、反噬、反控、重构规则。
- 面板字段是否稳定，奖励名称和数值是否统一。
