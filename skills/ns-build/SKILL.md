# NS Build

只搭结构和写作地图，不写正文，不维护日常进度。存在 `NOVEL.md` 时先读；写入需转 `$ns-canon` 并获同意。

## 写入

- YAML 写项目身份、规模计划、人物/世界摘要、初始连续性和风格契约。
- notes 写长人物档案、世界说明、时间线、术语表和风格样例。

## 类型模块

需要类型规则时读 [genre-modules.md](references/genre-modules.md)，只选相关模块。
多类型作品先定主类型，再选 1-2 个辅助模块；不要把所有类型都写进项目。

## 策略

- 短篇抓核心变化；中篇抓关键转折；长篇先做卷计划和近期章节。
- 系统文/无限流规则要有代价、限制和阶段变化。

## 规模规划

规模写 `plan.yaml.scale`，卷/番外计划写 `plan.yaml.volumes[]` 和 `extras[]`。卷简介写 `_index.md`，不复制进 `plan.yaml`。改卷计划时先改 YAML，再同步必要的人读目录。

## 输出

优先更新 YAML 和必要 notes。回复只概述关键决定、已更新文件和下一步入口。
