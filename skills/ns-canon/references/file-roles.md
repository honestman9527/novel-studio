# File Roles

用于判断某个 YAML 或 Markdown 文件是否该写、写到什么程度、哪些内容应转移到 notes 或 logs。

## 原则

- YAML 保存结构化事实、状态、索引和短摘要。
- Markdown 保存正文、人读说明、长说明、展示文案和过程记录。
- Frontmatter 只连接正文文件和 YAML 索引，不承载长说明。
- 根指导文件按 `AGENTS.md` / `CLAUDE.md` 查找；写入走 `$ns-guidance`。普通维护时没有目标文件就不主动新建，初始化时可生成基础模块。
- 当一个字段需要解释很多背景时，YAML 中保留摘要和 notes 路径。

## YAML 文件

| 文件 | 写什么 | 何时写 | 避免 |
| --- | --- | --- | --- |
| `project.yaml` | 项目身份、题材、受众、承诺、禁区摘要 | 开书、改定位、明确读者承诺 | 长设定、剧情大纲 |
| `plan.yaml` | 全书规模、卷计划、章节计划、番外计划、下一步 | 建框架、调规模、改卷计划 | 卷简介长文、完整大纲正文 |
| `memory.yaml` | 人物/世界/关系/名词/伏笔的结构化摘要 | 设定确认、章节产生新事实、结构改 | 人物长传、世界观论文 |
| `continuity.yaml` | 当前状态、已发生事件、未收束线、改写影响 | 写完章节、修订剧情、整理进度 | 正文摘要全文、未来正文 |
| `index.yaml` | 正文路径、排序、状态、字数和时间 | 新建/移动章节、字数审计后 | `content_root`、长说明 |
| `style.yaml` | 文风约束、章节结构契约、写法禁忌摘要 | 确认风格、修订风格规则 | 大段样文；样文进 notes |
| `research.yaml` | 来源摘要、待查问题、事实边界 | 调研后、来源冲突时 | 长摘录、原文搬运 |
| `art.yaml` | 视觉稳定要素、提示词文件路径、媒体索引 | 生成/整理视觉提示词或素材 | 完整提示词；全文进 `visuals/*.md` |
| `finish.yaml` | 完稿状态、里程碑、输出文件索引 | 分卷完成、全书完稿、交付物变化 | 正文或梗概全文 |
| `publish.yaml` | 正文根目录、排序、slug、封面、过滤规则 | 初始化发布规则、调整导出范围 | 章节列表详情；列表归 `index.yaml` |

## Markdown 文件

| 文件 | 写什么 | 何时写 | 避免 |
| --- | --- | --- | --- |
| `AGENTS.md` / `CLAUDE.md` | 根指导：必须遵守、不要写/改、风格偏好、内容边界、结构偏好、待确认 | 用户明确要维护全局指导，或初始化时需要基础模块 | 设定库、大纲、聊天记录；普通维护时没有文件就主动新建 |
| `content/volumes/*/_index.md` | 卷简介、卷承诺、主要人物、章节目录 | 新建卷、改卷承诺、同步章节目录 | 替代 `plan.yaml` 做计划源 |
| `content/volumes/*/ch*.md` | 章节 frontmatter、写作目标、正文、可选章末笔记 | 写新章、修订章节 | 把正文拆进 notes |
| `content/extras/*.md` | 番外或独立单元正文 | 写番外、序章、尾声、短篇 | 混入主线章节编号 |
| `brief.md` | 对外简介、标签、卖点、pitch、封面文案 | 做文案、投稿展示、对外介绍 | 内部长梗概；长梗概进 notes |
| `notes/*.md` | 人物档案、世界说明、时间线、术语、梗概、风格样例、待确认问题 | 说明变长、主题需要人读整理时 | 多主题混在一个超长文件 |
| `logs/*.md` | 调研、修订、迁移、决策过程记录 | 需要保留过程或操作痕迹时 | 代替 YAML 记录最终事实 |
| `visuals/*.md` | 完整视觉提示词、负面词、参数、迭代记录 | 生成或整理视觉提示词时 | 把稳定索引只放在 visuals，不同步 `art.yaml` |

## 同步提示

- 新建章节后，同步 `index.yaml` 和所属卷 `_index.md`。
- 修改正文后，先统计字数，再更新 frontmatter 和 `index.yaml`。
- 结构改影响事实时，同步 `continuity.yaml`、`memory.yaml`，必要时写 notes。
- 修改 notes 后，YAML 只同步摘要和路径。
