# NS Guidance

维护作者参与确认的 agent 项目约束；不写正文，不写设定库，不复制小说记忆或进度。

## 目标文件

按顺序寻找：`AGENTS.md`、`CLAUDE.md`。

- 找到后写入第一个目标；不要为短指导另建文件。
- 普通维护时两个都没有，不主动创建；给出建议文本并让作者选择目标。
- 项目初始化时可用 [agent-constraints.md](assets/templates/agent-constraints.md) 生成基础模块。
- 需要判断是否该写入或如何压缩时，读 [guidance-rules.md](references/guidance-rules.md)。

## 写入内容

- agent 工作流：先读哪些文件、何时确认、何时转技能、如何回写。
- 写入边界：不要碰的文件、不要新建的文件、需要人工确认的动作。
- 协作偏好：输出格式、询问方式、验证要求、工具使用习惯。

## 避免

- 不写人物长档、世界观长说明、大纲、进度、正文摘要、调研原文。
- 不把同一规则重复追加；先合并或改写已有条目。
- 不把临时修订意见写成 agent 约束，除非作者明确说以后都要。
- 小说记忆、剧情进度和过程记录转 `$ns-canon` 写入 `novel-studio/`。

## 流程

- 先询问或复述作者意图，避免替作者发明规则。
- 写入前压缩成可执行短句；一条只表达一个动作或边界。
- 能放进现有小节就更新现有小节；没有合适小节时再新增精简小节。
- 回复说明写入位置、保留了哪些待确认点。
