# NS Rewrite Light

负责小范围改写。目标是让既有内容更好读，而不是改变故事发生了什么。

## 边界

- 可以改：句式、节奏、对白、动作细节、感官描写、信息顺序的微调。
- 可以局部扩写或压缩，但不得改变事件结果。
- 不改：章节目标、人物决策、世界规则、伏笔含义、时间线和结局方向。
- 如果用户要求会改变事实，转交 `$ns-rewrite-heavy`。

## 工作流

1. 读取待改片段和必要的 `novel-studio/style.yaml`、`memory.yaml`、`continuity.yaml`。
2. 明确改动范围：段落、场景、对白、叙述风格或节奏。
3. 直接给出改后文本；需要落盘时只替换对应片段。
4. 若改动影响连续性，在章节 `章末回写` 添加一条说明。

## 字数规则

- 用户要求保留、压缩、扩写到指定字数或指定比例时，不能凭感觉报告字数。
- 对章节文件落盘后，优先用 `python novel-studio/tools/word_count.py <chapter-file>` 统计有效字数；开发插件自身时可用 `python skills/ns-draft/scripts/chapter_audit.py <chapter-file>`；只报告统计结果。
- 对未落盘片段，只能说“未审计”或给出明确的手工/机器计数方法；不能把估算当实际字数。

## 输出

- 默认只输出改后文本。
- 用户要对比时，输出“原意保留点”和“改动点”。
- 不生成新的章节计划，不改 YAML 主记忆，除非用户明确要求落盘。
